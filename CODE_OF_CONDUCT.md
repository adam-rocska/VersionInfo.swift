# Code of Conduct

## Standard

This project is for building high-quality Swift software.

Contributors are expected to:

- be direct without being abusive
- argue from facts, tests, and reproducible examples
- keep criticism focused on code, behavior, documentation, and design
- respect maintainers' time by keeping issues and pull requests scoped
- disclose security-sensitive details privately

Unacceptable behavior includes:

- harassment, threats, or personal attacks
- discriminatory language or behavior
- publishing private information without explicit permission
- knowingly filing noisy, misleading, or bad-faith issues
- pushing security exploit details into public issues or pull requests

## Enforcement

Maintainers may edit, hide, lock, or delete comments and issues that violate
this standard. Maintainers may also block participants who repeatedly make the
project harder to maintain.

If something needs maintainer attention, open a focused issue or use the private
security reporting path described in `SECURITY.md`.
