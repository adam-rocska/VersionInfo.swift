# Generated Version Information

Understand the `versions` value generated by `VersionInfoPlugin`.

## The Generated API

`VersionInfoPlugin` generates this shape in the consuming target:

```swift
typealias Version = (name: String, hash: String)
typealias Versions = (head: Version, heads: [Version], tags: [Version])

let versions: Versions
```

The names are intentionally plain. The generated file becomes part of your
target, so using it should feel like using code you wrote yourself.

## HEAD

`versions.head` describes the current checkout:

```swift
let head = versions.head

print("branch or checkout: \(head.name)")
print("commit: \(head.hash)")
```

When `HEAD` is symbolic, `name` is the branch name. When the repository is
detached, `name` is `"HEAD"`.

## Branches and Tags

`versions.heads` contains local branch refs.

`versions.tags` contains tag refs.

```swift
let branchNames = versions.heads.map(\.name)
let tagNames = versions.tags.map(\.name)
```

To print semantic tags, add the `VersionInfo` library product and parse the tag
refs:

```swift
import VersionInfo

for tag in versions.tags {
  guard let version = SemanticVersion(tag) else { continue }
  print("\(version) at \(tag.hash)")
}
```

## Repository Shapes

The generator reads Git metadata directly. It supports:

- loose refs
- packed refs
- `.git` files with `gitdir: ...`
- symbolic `HEAD`
- detached `HEAD`
- nested branch and tag names

The generated Swift file contains resolved values. Your application does not
need Git at runtime.

## Why a Tuple?

``Version`` is intentionally a tuple:

```swift
public typealias Version = (name: String, hash: String)
```

Plugin-only users should not need a runtime library dependency. The generated
source therefore carries the tuple shape directly.

The library exposes the same tuple shape so users who do add `VersionInfo` can
pass generated refs straight into ``SemanticVersion/init(_:)-(Version)``.
