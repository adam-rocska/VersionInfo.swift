# Getting Started

Add build-time Git version information to a Swift package target.

## Add the Package

Add VersionInfo.swift to your package dependencies:

```swift
dependencies: [
  .package(
    url: "https://github.com/adam-rocska/VersionInfo.swift",
    from: "1.0.0"
  )
]
```

## Attach the Plugin to a Target

Attach `VersionInfoPlugin` to the target that needs build information:

```swift
.executableTarget(
  name: "MyTool",
  plugins: [
    .plugin(name: "VersionInfoPlugin", package: "VersionInfo.swift")
  ]
)
```

This is a build setting, not a Swift import. SwiftPM runs the plugin before it
compiles `MyTool`, and the plugin generates a Swift source file for that same
target.

## Use the Generated Value

Read `versions` from normal Swift code in the target:

```swift
print("built from \(versions.head.name)")
print("commit \(versions.head.hash)")
```

The generated value is static Swift source. Once the target is built, no runtime
Git lookup is needed.

## Add the Library When Needed

If you also want semantic version parsing, add the library product:

```swift
.executableTarget(
  name: "MyTool",
  dependencies: [
    .product(name: "VersionInfo", package: "VersionInfo.swift")
  ],
  plugins: [
    .plugin(name: "VersionInfoPlugin", package: "VersionInfo.swift")
  ]
)
```

Then parse generated tags:

```swift
import VersionInfo

let semanticTags = versions.tags.compactMap(SemanticVersion.init)
let latest = semanticTags.max()
```

The tuple generated by the plugin intentionally matches ``Version``. That is
why `SemanticVersion.init(_:)` can parse a generated tag directly.

## What to Expect

The plugin captures the Git state visible to SwiftPM at build time. It declares
the visible Git metadata files as build inputs, so changed branch, tag, packed
ref, and `HEAD` files are picked up by SwiftPM's build planning.

Use the plugin for build identity. Use the library for SemVer behavior. Use both
when your Git tags are semantic versions.
